默认您已经安装好了LAMP!

步骤1：为WordPress创建MySQL数据库和用户

我们将采取的第一步是准备工作。WordPress使用MySQL来管理和存储站点和用户信息。我们已经安装了MySQL，但是我们需要创建一个数据库和一个供WordPress使用的用户。 首先，通过发出以下命令登录到MySQL根（管理）帐户：

mysql -u root -p

安装该软件时，系统将提示您输入为MySQL根帐户设置的密码。 首先，我们可以创建一个可以由WordPress控制的独立数据库。 您可以随意调用此名称，但wordpress在本指南中我们将使用它来简化操作。您可以通过键入以下内容为WordPress创建数据库：

mysql> CREATE DATABASE wordpress DEFAULT CHARACTER SET utf8 COLLATE utf8_unicode_ci;

接下来，我们将创建一个单独的MySQL用户帐户，该帐户将专用于我们的新数据库。从管理和安全的角度来看，创建单一功能的数据库和帐户是一个好主意。 我们将wordpressuser在本指南中使用该名称。如果需要，可以随时更改此设置。 我们将创建该帐户，设置密码，并授予对我们创建的数据库的访问权限。我们可以通过键入以下命令来做到这一点。请记住在这里为您的数据库用户选择一个强密码：

mysql> GRANT ALL ON wordpress.* TO 'wordpressuser'@'localhost' IDENTIFIED BY 'password';

现在，您有一个数据库和用户帐户，每个帐户和帐户都是专门为WordPress创建的。我们需要清除特权，以便MySQL的当前实例知道我们最近所做的更改：

mysql> FLUSH PRIVILEGES;

输入以下内容退出MySQL：

mysql> EXIT;

步骤2：安装其他PHP扩展

设置我们的LAMP堆栈时，我们只需要很少的扩展集即可使PHP与MySQL进行通信。WordPress及其许多插件利用了其他PHP扩展。 我们可以通过键入以下内容来下载并安装一些最流行的PHP扩展以用于WordPress：

sudo apt-get update
sudo apt-get install php-curl php-gd php-mbstring php-mcrypt php-xml php-xmlrpc

在下一部分中，我们将重新启动Apache以利用这些新扩展。如果返回此处以安装其他插件，则可以通过键入以下内容立即重新启动Apache：

sudo systemctl restart apache2

步骤3：调整Apache的配置以允许.htaccess覆盖和重写

接下来，我们将对Apache配置进行一些小的调整。当前，.htaccess禁止使用文件。WordPress和许多WordPress插件广泛使用这些文件对Web服务器的行为进行目录内调整。 此外，我们将启用mod_rewrite，这是使WordPress永久链接正确运行所必需的。

  • 启用.htaccess替代

打开主要的Apache配置文件以进行我们的第一个更改, 要允许.htaccess文件，我们需要AllowOverride在Directory指向文档根目录的块中设置指令。在文件底部，添加以下块：

sudo nano /etc/apache2/apache2.conf
. . .

<Directory /var/www/html/>
    AllowOverride All
</Directory>

. . .

完成后，保存并关闭文件。

  • 启用重写模块

接下来，我们可以启用mod_rewrite以便可以利用WordPress永久链接功能：

sudo a2enmod rewrite
  • 启用更改

在实施所做的更改之前，请检查以确保没有发生任何语法错误：

sudo apache2ctl configtest

输出中可能会显示一条消息，如下所示：

AH00558: apache2: Could not reliably determine the server's fully qualified domain name, using 127.0.1.1. Set the 'ServerName' directive globally to suppress this message
Syntax OK

如果要取消显示第一行，只需ServerName在/etc/apache2/apache2.conf文件中添加一个指向服务器域或IP地址的指令即可。 但是，这只是一条消息，并不影响我们网站的功能。只要输出包含Syntax OK，您就可以继续。 重新启动Apache以实施更改：

sudo systemctl restart apache2

步骤4：下载WordPress

现在我们的服务器软件已配置完毕，我们可以下载并设置WordPress。特别是出于安全原因，始终建议从其站点获取最新版本的WordPress。 切换到可写目录，然后通过键入以下命令下载压缩版本：

cd /tmp
curl -O https://wordpress.org/latest.tar.gz

解压缩压缩文件以创建WordPress目录结构：

tar xzvf latest.tar.gz

我们将暂时将这些文件移到我们的文档中。在执行此操作之前，我们可以添加一个虚拟.htaccess文件并设置其权限，以便WordPress可以稍后使用它。 创建文件并通过键入以下内容设置权限：

touch /tmp/wordpress/.htaccess
chmod 660 /tmp/wordpress/.htaccess

我们还将复制示例配置文件到WordPress实际读取的文件名：

cp /tmp/wordpress/wp-config-sample.php /tmp/wordpress/wp-config.php

我们还可以创建upgrade目录，以使WordPress在更新其软件后尝试自行执行此操作时不会遇到权限问题：

mkdir /tmp/wordpress/wp-content/upgrade

现在，我们可以将目录的全部内容复制到文档根目录中。 我们正在使用该-a标志来确保我们的权限得到维护。我们在源目录的末尾使用一个点来指示应复制目录中的所有内容，包括隐藏文件（如.htaccess我们创建的文件）：

sudo cp -a /tmp/wordpress/. /var/www/html

步骤5：配置WordPress目录

在进行基于Web的WordPress设置之前，我们需要调整WordPress目录中的某些项目。 调整所有权和权限 我们需要完成的一大工作是设置合理的文件权限和所有权。 我们需要能够以普通用户的身份写入这些文件，并且我们还需要Web服务器能够访问和调整某些文件和目录才能正常运行。

首先，我们将文档根目录中所有文件的所有权分配给用户名。 sammy在本指南中，我们将使用我们的用户名作为用户名，但是您应该更改它以匹配您的sudo用户名。我们将组所有权分配给该www-data组：

sudo chown -R sammy:www-data /var/www/html

接下来，我们将setgid在文档根目录内的每个目录上设置该位。 这将导致在这些目录中创建的新文件继承父目录的组（我们刚刚将其设置为www-data），而不是创建用户的主组。 这只是确保只要我们在命令行的目录中创建文件，Web服务器仍将对该文件拥有组所有权。

我们可以setgid通过键入以下内容来设置WordPress安装中每个目录的位：

sudo find /var/www/html -type d -exec chmod g+s {} \;

我们还会调整其他一些细粒度的权限。首先，我们将授予组对wp-content目录的写入权限，以便Web界面可以更改主题和插件：

sudo chmod g+w /var/www/html/wp-content

作为此过程的一部分，我们将为Web服务器赋予对这两个目录中所有内容的写访问权限：

sudo chmod -R g+w /var/www/html/wp-content/themes
sudo chmod -R g+w /var/www/html/wp-content/plugins

首先应该设置合理的权限。一些插件和过程可能需要进行其他调整。

设置WordPress配置文件 现在，我们需要对主WordPress配置文件进行一些更改。

当我们打开文件时，我们的首要任务是调整一些秘密密钥，以为我们的安装提供一些安全性。 WordPress为这些值提供了安全的生成器，因此您不必自己尝试提出好的值。 这些仅在内部使用，因此在这里拥有复杂，安全的值不会损害可用性。

要从WordPress密钥生成器获取安全值，请输入：

curl -s https://api.wordpress.org/secret-key/1.1/salt/

接下来，我们需要在文件开头修改一些数据库连接设置。您需要调整数据库名称，数据库用户以及我们在MySQL中配置的关联密码。 我们需要做的另一项更改是设置WordPress应该用于写入文件系统的方法。 由于我们已授予Web服务器写入所需位置的权限，因此我们可以将文件系统方法显式设置为“直接” 。如果无法使用我们当前的设置进行设置，则当我们执行某些操作时，WordPress会提示您输入FTP凭据。

可以在数据库连接设置下面或文件的其他任何位置添加此设置：

/var/www/html/wp-config.php
. . .

define('DB_NAME', 'wordpress');

/** MySQL database username */
define('DB_USER', 'wordpressuser');

/** MySQL database password */
define('DB_PASSWORD', 'password');

. . .

define('FS_METHOD', 'direct');

完成后保存并关闭文件。

步骤6：通过Web界面完成安装

现在服务器配置已完成，我们可以通过Web界面完成安装。

在Web浏览器中，导航到服务器的域名或公共IP地址：

http://server_domain_or_IP
